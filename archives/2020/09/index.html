数据仓库
原创 | 2020-09-18 | Eric

数据仓库一、简介数据仓库（Data Warehouse）简称DW或DWH，是数据库的一种概念上的升级，可以说是为满足新需求设计的一种新数据库，而这个数据库是需容纳更多的数据，更加庞大的数据集，从逻辑上讲数据仓库和数据库是没有什么区别的。为企业所有级别的决策制定过程，提供所有类型数据支撑的战略集合，主要是用于数据挖掘和数据分析，以建立数据沙盘为基础，为消灭消息孤岛和支持决策为目的而创建的。 二、数据仓库分层1. ODS层（原始数据层）存放原始数据，直接加载原始日志、数据，数据保持原貌不做处理 2. DWD层（明细数据层）结构和粒度与原始表保持一致，对ODS层数据进行清洗（去除空值，脏数据，超过极限范围的数据) 3. DWS层（服务数据层）进行轻度汇总 4. ADS层（数据应用层）为各个统计报表提供数据 为什么要进行分层(1) 把复杂的问题简单化：将一个复杂的任务分解成多个步骤来完成，每一层只处理单一步骤，比较简单、并且方便定位问题 (2) 减少重复开发：规范数据分层，通过的中间层数据，能够减少极大的重复计算，增加一次计算结果的复用性。 (3) 隔离原始数据：不论是数据的异常还是数据的敏感性，都使真实数据与统计数据解耦开。 二、数据仓库与数据库的区别1. 数据来源数据库的数据来源于业务，数据仓库的数据来自于多个数据源 2. 时间维度数据库保留的是当前的数据，数据仓库保留的数据是时间轴上的全部信息 3. 组织方式数据库面向业务组织，数据仓库面向主题 4. 建模方式数据库服从范式建模，数据仓库一般是合理冗余的 5. 设计目的数据库以存储、管理为主，数据仓库以组织、计算为主 三、数据仓库的存储方式1. 全量存储没有分区，会直接覆盖 2. 增量数据每个分区只存储当日从数据库表中增量抽取出的数据 3. 快照存储每个分区相当于一个全量表 4. 拉链存储存储了数据的变化情况 四、数仓的模型1. 星型模型2. 雪花模型3. 星座模型对比：雪花模型是将星型模型的维度进一步划分，使各个维度表满足规范化设计。而星座模型则是允许星形模式中出现多个事实表。
